## 🔹 Variables & Data Types

Variables are used to store values in Python.

Python supports several built-in data types such as:

- Integer
- Float
- Boolean
- NoneType
- String

These data types allow developers to store and manipulate different types of data in programs.

---

## 🔹 Input Function

Python provides the `input()` function which allows users to provide input during program execution.

User input is commonly used for:
- interactive programs
- dynamic data processing
- accepting values from users

---

## 🔹 Strings in Python

A string is a sequence of characters used to represent text.

Concepts covered in strings:
- Creating strings
- String indexing
- String slicing
- Negative indexing

Strings are widely used for handling textual data in Python programs.

---

## 🔹 Escape Sequences

Escape sequences are special characters used to format text within strings.

Common escape sequences include:

- `\n` → New line
- `\t` → Tab space
- `\"` → Double quotation mark
- `\'` → Single quotation mark
- `\\` → Backslash

These sequences help improve output formatting.

---

## 🔹 Lists in Python

A list is a collection data type used to store multiple values in a single variable.

Important properties of lists:
- Lists are **mutable** (changeable)
- Lists can store **multiple data types**
- Lists allow **indexing and slicing**

List operations practiced include:
- Adding elements
- Removing elements
- Sorting elements
- Accessing list items

---

## 🔹 Tuples in Python

A tuple is similar to a list but has one major difference.

Key properties of tuples:
- Tuples are **immutable**
- Once created, tuple values cannot be changed
- Tuples are useful for storing fixed collections of data

---

## 🔹 External Python Module

Python allows the use of external modules and libraries to extend functionality.

One external module explored in this learning phase is:

**pyttsx3**

This module is used for **Text-to-Speech (TTS)** functionality, allowing Python programs to convert text into spoken audio.

File handling allows Python programs to **read from and write data to files**.

This is useful for:
- Saving program data
- Reading stored information
- Updating files dynamically
- Storing game results or logs

---

## 🔹 File I/O Workflow

The basic workflow of file handling in Python includes:

1. Opening a file  
2. Performing operations (read/write/update)  
3. Closing the file  

Proper file handling helps in **managing data efficiently and preventing data loss**.

---

## 🔹 File Opening Modes

Different file modes were explored during this learning phase:

| Mode | Description |
|-----|-------------|
| `r` | Read mode |
| `w` | Write mode |
| `a` | Append mode |
| `r+` | Read and write |
| `w+` | Write and read |
| `a+` | Append and read |

Understanding these modes is important when working with persistent data.

# 📚 Day 14 – Object-Oriented Programming (Part 1)

On Day 14, I learned the foundational concepts of **Object-Oriented Programming**.

This included understanding how Python uses **classes and objects** to represent real-world entities in code.

Concepts explored during this phase included the creation of classes, object instantiation, the role of instance variables, and the use of the `self` keyword to reference object attributes.

I also learned how constructors work in Python and how they help initialize object data when an object is created.

Another concept explored was the use of **static methods**, which allow functions to exist within a class without requiring access to instance-specific data.

Several practice problems were implemented to strengthen the understanding of these concepts.

---

# 📚 Day 15 – Object-Oriented Programming (Part 2)

Day 15 focused on deeper OOP concepts and how classes can interact with each other through **inheritance**.

Inheritance allows new classes to reuse properties and behavior from existing classes, improving code reusability and maintainability.

Different inheritance models were explored, including **single inheritance, multiple inheritance, and multilevel inheritance**.

Additional concepts covered included the use of **class methods**, which allow access to class-level data, and **property decorators**, which help manage attribute access in a controlled way.

I also learned how **getter and setter methods** can be implemented to control how class attributes are read or modified.

Another concept explored was **method overloading**, which helps implement flexible behavior in class methods.

Practice exercises were solved to apply these concepts in real programming scenarios.
